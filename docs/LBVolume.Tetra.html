Class: Tetra

LBVolume.Tetra

new Tetra(verticesopt, massopt, indicesopt) → {LBVolume.Tetra}

Represents a tetrahedron. The four vertices of the tetra are ordered such that v[0],v[1],v[2] are CCW about the face (right hand rule), as are v[0],[v2],v[3].

(static) cuboidToTetras(vertexIndices, vertexArray, tetrasopt) → {Array.<LBVolume.Tetra>}

Converts a set of 8 vertices describing a cuboid (a convex polyhedron with 6 quadrilateral faces) into an array of 6 tetrahedra representing the volume.

This is currently a fairly simple algorithm, it does not check for vertices lying on each other, and expects the vertices in a particular order.

The vertices should be specified such that the first 4 vertices are one face, with the order in the CCW direction (right-hand rule), while the second 4 vertices are the opposite face, with the opposite order so that v[4] is connected to v[0], v[5] to v[1], v[6] to v[2], and v[7] to v[3].

(static) triangularBipyramidToTetras(vertexIndices, vertexArray, tetrasopt) → {Array.<LBVolume.Tetra>}

Converts a set of 5 vertices describing a triangular bipyramid into an array of two tetrahedra representing the volume.

This is currently a fairly simple algorithm, it does not check for vertices lying on each other, and expects the vertices in a particular order.

The bipyramid must be described such that the first four vertices describe one tetrahedron, and the last four vertices describe the other tetrahedron (the face v[1], v[2], v[3] is internal and does not exist).

(static) triangularPrismToTetras(vertexIndices, vertexArray, tetrasopt) → {Array.<LBVolume.Tetra>}

Converts a set of 6 vertices describing a triangular prism into an array of three tetrahedra representing the volume.

This is currently a fairly simple algorithm, it does not check for vertices lying on each other, and expects the vertices in a particular order.

The prism must be described such that the first 3 vertices represent one triangular face and the second 3 vertices the other triangular face, and v[0] is connected to v[3], v[1] is connected to v[4], and v[2] is connected to v[5].
